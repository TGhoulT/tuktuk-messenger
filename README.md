# Тук-Тук – защищённый веб-мессенджер

## Описание

«Тук-Тук» – это веб-мессенджер с серверным шифрованием (AES‑256‑GCM), аутентификацией по JWT и системой личных диалогов. Реализован на стеке **Node.js (Express) + React (TypeScript)** с использованием **PostgreSQL** и **Socket.IO** для мгновенной доставки сообщений.
### Реализованный функционал

- Регистрация и вход (JWT + refresh‑токены)
- Личные чаты (диалоги) с временными чатами до первого сообщения
- Отправка, редактирование, удаление, пересылка и закрепление текстовых сообщений
- Реакции, поиск по сообщениям и пользователям
- Настройки приватности, онлайн-статус, индикатор печати
- Светлая / тёмная тема, две версии отображения сообщений (Z и V)
- Адаптивная вёрстка, PWA-поддержка (установка на устройство)

### Планы развития (фронтенд)

- Создание групп и каналов (бэкенд полностью готов)
- Отправка файлов, изображений, стикеров, GIF и голосовых сообщений (бэкенд готов)
- Загрузка собственных тем оформления (бэкенд готов)
- Экспорт/импорт тем

> **Бэкенд полностью поддерживает группы, каналы, отправку файлов, стикеров и GIF, но интерфейс этих функций пока не реализован – это следующий этап разработки.**

## Установка и запуск
### 1. Клонирование репозитория

```bash
git clone https://github.com/TGhoulT/tuktuk-messenger.git
cd tuk-tuk
```

### 2. Настройка бэкенда

Перейдите в папку `backend` и установите зависимости:

```bash
cd backend
npm install
```

Создайте файл `.env` из примера:

```bash
cp .env.example .env
```

Отредактируйте `.env`, указав свои настройки базы данных, секретные ключи и порты.

Запустите миграции базы данных:

```bash
npm run migration:run
```

Запустите сервер в режиме разработки:

```bash
npm run dev
```

Сервер будет доступен по адресу `http://localhost:3000`.

### 3. Настройка фронтенда

Откройте новый терминал, перейдите в папку `frontend`:

```bash
cd ../frontend
npm install
```

Создайте файл `.env` из примера:

```bash
cp .env.example .env
```

При необходимости измените `VITE_API_URL` на адрес вашего бэкенда.

Запустите фронтенд в режиме разработки:

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173` (или другому, указанному в терминале).

## Важные замечания

- Для корректной работы шифрования убедитесь, что в `.env` заданы **MASTER_KEY** и **MASTER_SALT** (генерируются случайно один раз).
    
- Для продакшена измените значения **JWT_ACCESS_SECRET** и **JWT_REFRESH_SECRET** на длинные случайные строки.
    
- Все секретные данные хранятся в `.env` и **не попадают** в репозиторий (игнорируются через `.gitignore`). Файлы `.env.example` (шаблоны) коммитятся, чтобы разработчики знали, какие переменные нужны.
